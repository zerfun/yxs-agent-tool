# 研享数Agent工具 - 架构设计文档

## 系统概述

研享数Agent工具是一个跨端AI Agent控制系统，允许用户通过移动端通讯应用（微信、飞书、QQ等）远程控制电脑端的AI Agent，支持多个AI模型的集成。

## 模块设计

### 1. API网关层 (FastAPI)

**职责:**
- 接收来自各平台的webhook回调
- 验证请求签名和身份
- 将消息转发给相应的处理器
- 返回标准化的API响应

**主要端点:**
- `POST /api/v1/wechat/callback` - 微信消息回调
- `GET /api/v1/wechat/callback` - 微信服务验证
- `POST /api/v1/feishu/callback` - 飞书消息回调
- `POST /api/v1/agent/task` - 创建任务
- `GET /api/v1/agent/task/{task_id}` - 查询任务状态

### 2. 消息处理层

#### 2.1 微信处理器 (WeChatService)

**功能:**
- 解析XML格式的微信消息
- 验证消息签名
- 生成XML格式的回复消息
- 处理微信事件（关注、取消关注等）

**关键方法:**
```python
- parse_message(data: str) -> Dict  # 解析微信消息
- verify_signature(sig, ts, nonce) -> bool  # 验证签名
- create_response(from, to, content) -> str  # 生成回复
```

#### 2.2 飞书处理器 (FeishuService)

**功能:**
- 解析JSON格式的飞书消息
- 验证飞书请求
- 调用飞书API发送回复

#### 2.3 QQ处理器 (QQService)

**功能:**
- 集成QQ机器人API
- 处理QQ消息

### 3. 任务处理层 (AgentService)

**职责:**
- 创建和管理任务
- 根据prompt和模型类型调用相应AI API
- 处理任务异常
- 保存执行结果

**主要方法:**
```python
process_task(request: TaskRequest, user_id: str, source: MessageSource) -> TaskResponse
_call_codex(prompt: str, ...) -> str
_call_claude(prompt: str, ...) -> str
_call_qwen(prompt: str, ...) -> str
get_task(task_id: str) -> Optional[TaskResponse]
```

### 4. AI模型适配层

#### 4.1 Codex适配器
- 集成GitHub Copilot / Codex API
- 代码生成、补全、解释

#### 4.2 Claude适配器
- 集成Anthropic Claude API
- 自然语言处理、代码理解

#### 4.3 Qwen适配器
- 集成阿里云Qwen API
- 中文优化、代码生成

### 5. 数据持久化层

#### 5.1 MongoDB
- 存储任务记录
- 存储用户信息
- 存储会话历史

#### 5.2 Redis
- 缓存任务结果
- 会话管理
- 速率限制

### 6. 消息队列层

#### 6.1 Celery/Redis
- 异步任务处理
- 任务去重
- 重试机制
- 优先级队列

## 数据流

### 场景1: 用户通过微信发送代码生成请求

1. 用户在微信中发送: "@Agent 帮我写一个快速排序的Python函数"
2. 微信服务器将消息转发到 `/api/v1/wechat/callback`
3. FastAPI接收请求，调用WeChatService解析消息
4. WeChatService验证签名并解析XML
5. 将prompt转发给AgentService创建任务
6. AgentService调用Codex API
7. Codex返回代码
8. 结果保存到数据库
9. 生成XML格式回复
10. 通过微信API发送给用户

### 场景2: 用户查询任务状态

1. 用户发送: "@Agent 查询上一个任务的进度"
2. 解析意图，调用 `/api/v1/agent/task/{task_id}`
3. 从Redis缓存或MongoDB查询任务状态
4. 返回给用户

## 扩展性设计

### 1. 模块化架构
- 每个平台独立的Service类
- 每个AI模型独立的适配器
- 易于添加新平台或模型

### 2. 异步处理
- 使用异步函数处理IO操作
- Celery处理长时间运行的任务
- 提高系统吞吐量

### 3. 缓存策略
- 频繁访问的数据缓存到Redis
- 任务结果缓存24小时
- 用户信息缓存

### 4. 监控和日志
- 详细的操作日志
- 错误追踪
- 性能监控
- 告警机制
